Detailed Description

Storage for geometric objects with unified identification system.

Usage Example

GeometryStorage storage;
// Create points
auto [id1, p1] = storage.createPoint(0.0, 5.0);
auto [id2, p2] = storage.createPoint(10.0, 5.0);
// Create line between points
auto [lineId, line] = storage.createLine(p1, p2);
// Get object by ID
Point2D* point = storage.get<Point2D>(id1).value();
// Safe retrieval with type checking
auto result = storage.get<Line2D>(lineId);
if (result) {
Line2D* myLine = result.value();
}

◆ remove()

void OurPaintDCM::Figures::GeometryStorage::remove ( ID  id,
bool  forceCascade = false 
)

Remove object by ID.

Parameters
idIdentifier of object to remove
forceCascadeIf true — also removes dependent objects. If false — returns DEPENDENCY_EXISTS error if dependencies exist.
Exceptions
std::runtime_errorwhen ID not found or dependencies block removal

Algorithm

  1. Check existence
  2. If Point2D, check dependencies (Line/Circle/Arc that use it)
  3. If forceCascade=false and dependencies exist — error
  4. If forceCascade=true — first remove dependent objects
  5. Mark slot in deque as "deleted" (lazy deletion)

Why lazy deletion?

Real deletion from middle of deque is O(n). We just remove entry from index. Object remains in memory, but is inaccessible. Can implement compaction if needed.

std::deque<Point2D> OurPaintDCM::Figures::GeometryStorage::m_points
private

Storage for each object type.

We use deque instead of vector because:

  • On push_back, pointers to EXISTING elements are NOT invalidated
  • This is critical since Line/Circle/Arc store Point2D*
  • Vector on reallocation would move all Point2D, breaking all pointers
